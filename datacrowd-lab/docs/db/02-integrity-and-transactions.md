# Integrity & Transactions

## Как обеспечивается целостность
- PK/FK (внутри одной БД)
- NOT NULL / UNIQUE / CHECK ограничения
- транзакции на операциях изменения данных (на уровне сервисов через Spring)

## Почему нет FK между сервисами
Микросервисная изоляция: auth/core/payments — разные bounded context, поэтому межсервисные идентификаторы хранятся как UUID без FK.

## Где важны транзакции
- регистрация/логин (auth)
- создание проектов/датасетов/тасок (core)
- проведение оплаты и запись payment+ledger (payments)
